设计模式

发表于 2021-08-12
  • 大大大大前端
首页 计算机基础

# 1. 工厂模式

将重复的代码写成一个工厂函数，在调用工厂来创建对象或者返回一些其他东西

# 2. 建造者模式

把较复杂的构造函数的复杂逻辑拆分抽离成一个个独立的属性类，让我们的代码更加清晰，更具有维护性

# 3. 单例模式

单例模式就是希望可以再使用方法创建对象的时候，我们无论创建了多少次对象都指向同一个

# 4. 装饰器模式

装饰器模式希望在不改变源对象的基础上，通过对其扩展功能和属性来实现更复杂的逻辑

# 5. 组合模式

组合模式作用于将多个部分通过组合变成一个整体，就如果去 kfc 点了一个汉堡，两个薯条和已被可乐，我们把这些东西看成是一个个部分或者说是组件，通过组合就可以将整个套餐产出给顾客，就是组合模式

# 6. 观察者模式

观察者模式也叫发布订阅模式，是设计模式中非常著名也是非常重要的一种模式，这种模式一般会定义一个主体和众多的个体，这个主体可以想象成一个消息中心，里面有各种各样的消息，众多的个体可以订阅不同的消息，当未来消息中心发布某条消息的时候，订阅过他的个体就会得到通知

# 7. 策略模式

策略模式主要是用多种状态或者策略需要进行选择时，将所有选择封装在一起，只给外部暴露必要的接口

# 8. 链模式

链模式主要是实现链式调用用的，通过在自身方法中返回自身的方式，在一个对象连续多次调用自身方法，可以简化写法

# 9. 委托模式

委托模式希望当多个对象需要处理同一个请求时，可以将这些请求交由另一个对象同一处理

# 10. 数据访问对象模式

数据访问对象模式主要是用来抽象和封装一个对象来对数据源进行访问和存储，这样可以方便对数据的管理，以及避免数据间的重复，覆盖等问题

# 11.MVC 模式

MVC 是由 model - view - controller 组成的，它是一种使用业务逻辑，数据，视图进行分离的方式来组织架构的一种模式

# 12. MVVM 设计模式

MVVM 模式在传统的 MVC 模式下进行了改造，实现的重点在数据驱动视图的一种设计模式，详见 Vue 全家桶知识 [0]
