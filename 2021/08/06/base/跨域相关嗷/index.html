# 跨域（非同源策略请求）

  • 同源策略请求 ajax /fetch
  • 跨域传输

同源策略：指两个地址协议，域名，端口号都一样，只要有一个不同，就是跨域

解决跨域的方法：

# 1.JSONP

html 标签中的 script，img，link，iframe 等不存在跨域请求的限制

利用这一性质，在客户端的 script 的 src 下写一个服务器的地址，并且传一个回调函数 callback，在服务器中接收到客户端的请求（参数，和 callback）然后准备数据，客户端返回一个带有 callback 的字符串，浏览器拿到数据后，把返回来的字符串变成 js 表达式进行执行就能拿到服务端的数据

JSONP 存在的问题：

  • JSONP 只能处理 GET 请求
  • 需要后台服务器的支持
  • 如果别人对 url 进行劫持，数据会泄露，不安全

# 2.CORS 跨域资源共享

客户端正常发请求

服务端要用一个中间件，通过设置响应头来允许哪个源，哪些请求方式，哪些请求头跨域
