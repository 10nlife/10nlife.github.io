# Cookie 简单了解

# 1. cookie 作用

HTTP Cookie（Web Cookie）是服务器发送到用户浏览器并保存到本地大的一小块数据，他会在浏览器下次向同一服务器发起请求时被携带并发送到服务器上，它用于告知服务端两个请求是否来自同一服务器，如保持用户登陆状态，Cookie 使基于无状态的 HTTP 协议记录稳定信息成为了可能

作用：

  • 会话状态的管理（如用户登录状态，购物车，游戏分数或其他需要记录的信息）
  • 个性化设置（如用户自定义设置，主题等）
  • 浏览器行为跟踪（如跟踪分析用户行为等）

# 2. cookie 创建

当服务器收到 HTTP 请求时，服务器可以在响应首部里添加一个 Set-Cookie 选项，浏览器收到响应后通常会保存下 cookie，之后对该服务器每一次请求中都通过 cookie 请求首部将 cookie 信息发送到服务器，cookie 的过期时间，域，路径，有效期，适用站点都可以根据需要来指定

# 3. cookie 类型

  • 会话期的 cookie：存放在浏览器的内存里面，关闭浏览器就无了，生命周期就是浏览器的生命周期，不需要指定（Expires）或（Max-Age）
  • 持久性的 cookie：和关闭浏览器便失效的会话期 cookie 不同，持久性 cookie 可以指定一个特点的过期时间（Expires）或者有效期（Max-Age）
  • 第三方 cookie：如果 cookie 的域和页面的域不一样，则称之为第三方 cookie，用于广告和网路追踪

# 4. cookie 的属性

Secure 和 HttpOnly 属性

  • Secure：标为 Secure 的 Cookie 只应通过被 HTTPS 协议加密过的请求发送给服务器，但即便设置了 Secure 标记，敏感信息也不可以通过 Cookie 传输，不安全
  • HttpOnly：为了避免 XSS 攻击，通过 JavaScript 的 Document.cookie API 无法访问带有 HttpOnly 标记的 Cookie，他们只应该发送给服务端，如果 cookie 不想被脚本调用，那么为其设置 HttpOnly 属性

作用域属性

Domain 和 Path 标识定义了 cookie 的作用域：即 cookie 应该发送给哪些 URL

  • Domain：标识指定了哪些主机可以接受 cookie，如果不指定，默认为当前文档主机（不包含子域名），如果指定了 Domain，则一般包含子域名

  • Path：标识指定了主机下的哪些路径可以接受 cookie（该路径必须存在于请求 URL 中），例：设置了 Path = /docs，则 /docs/Web，/docs/Web/Http 都会被匹配

SameSite 属性

SameSite Cookie 允许服务器要求某个 cookie 在跨站请求时不会被发送，从而可以阻止跨站请求伪造攻击（CSRF）

SameSite 的值有：

  • None：浏览器会在同站请求，跨站请求下发送 cookies，不区分大小写
  • Strict：浏览器只发送同站请求的 cookie（当前网页 URL 与请求目标 URL 完全一致），如果请求来自与当前 location 的 URL 不同的 URL，则不包括标记为 Strict 属性的 cookie
  • Lax：默认选项，Same-site cookies 将会为一些跨站子请求保留，如图片加载或者 frames 的调用，但只有当用户从外部站点导航到 URL 时才会发送（link 链接）

Expires，MaxSize 属性（有效时间）
